## Qu'est-ce que l'architecture React exactement ?

Contrairement à d'autres frameworks et bibliothèques, ReactJS a un modèle d'architecture flexible.

C'est juste une vue qui est allouée aux besoins de l'interface utilisateur.

Sous l'interface utilisateur, il existe de nombreux composants React ; une petite unité structurelle - comme une étiquette, un bouton, une étiquette de
texte d'entrée ou quelque chose d'aussi complexe qu'un formulaire d'inscription, etc.

Et ce n'est même pas sa meilleure fonctionnalité : vous pouvez attribuer un état à chaque composant React ; cette fonctionnalité est facultative et
dépend du développeur. L'état peut être défini comme les données obligatoirement requises pour que l'application React fonctionne.

Ainsi, quelle que soit l'action entreprise par l'utilisateur, cela correspondra au changement nécessaire de l'état de temps en temps.

Cela signifie que quel que soit l'état actuel des données, déterminera les éléments affichés sur l'interface utilisateur de l'application. C'est
simplement ainsi que fonctionne l'architecture React.

## Avantages de l'architecture React pour la création d'applications de niveau entreprise

![React Architecture Advantages for building large Apps ](https://www.communicationcrafts.com/wp-content/uploads/2021/06/CC-Blog_React-Architecture-Best-Practices-and-Tips-from-Experts_inner-banner_1.jpg)

L'architecture React js est l'une des meilleures du lot pour créer des applications Web à grande échelle. Voici pourquoi :
- **Un état global cohérent** - Toutes les données de l'application peuvent être stockées dans l'état global. Les modifications apportées à cet état
apporteront automatiquement des modifications au reste des vues.
- **Évolutivité élevée** - L'architecture ReactJS peut être utilisée pour gérer un trafic important et répondre aux besoins d'une entreprise de niveau
entreprise lorsqu'elle est utilisée avec un framework backend évolutif comme NodeJS.
- **Tests faciles** - il est beaucoup plus facile pour les développeurs de tester des parties individuelles de l'application React en raison de ses
fonctionnalités et composants indépendants.
- **Composants découplés** - Redux-saga est utilisé par React pour lier les composants découplés ensemble. Cela vous permet d'effectuer de meilleurs
tests grâce à la prise en charge des flux asynchrones.

## Meilleures pratiques d'architecture React - Selon les experts de l'industrie

Nous ne saurions trop insister sur ce fait ; il est essentiel de suivre les meilleures pratiques de l'architecture React pour créer une grande
application qui résiste à l'épreuve du temps.

De l'utilisation des normes de codage de haute qualité à l'organisation de la structure des dossiers pour différentes équipes, si vous nous le demandez,
les meilleures pratiques ne verront jamais la fin de la lumière.

Certains développeurs aimeraient donner des noms dont ils se souviennent à leurs fichiers, tandis que pour d'autres, il pourrait s'agir d'utiliser les
meilleures bibliothèques ReactJS pour améliorer et augmenter l'efficacité de l'application.

**Voici les meilleures pratiques à suivre en utilisant l'architecture React :**

### Évitez d'utiliser beaucoup de fichiers et de dossiers imbriqués

Certains développeurs peuvent croire que l'attribution de balises utiles aux fichiers est la pratique la plus efficace en ce qui concerne les
applications Web React, tandis que d'autres peuvent croire que l'optimisation du flux de travail fonctionne mieux en utilisant les bibliothèques
idéales.

Nous avons mené des recherches approfondies et des discussions avec des experts de l'industrie pour apprendre les meilleures pratiques de React
Architecture pour la création et la mise à l'échelle d'applications Web de niveau entreprise. Voici ce que nous avons découvert !

### Réutilisez vos composants

La meilleure façon d'utiliser React est de développer un composant pour chaque fonction, cela vous permet de l'utiliser à plusieurs reprises. Cela
signifie que si vous avez besoin de créer un nouveau composant pour une fonction qui est la même qu'avant, utilisez-le au lieu de créer le nouveau.

Si la taille du composant devient importante, il est préférable de le diviser en composants plus petits.

La réutilisation des composants React dans votre projet ou dans plusieurs projets que vous développez vous aidera à atteindre une meilleure fiabilité.

### Mettez CSS dans JavaScript

Lorsque vous démarrez votre projet React, les experts de l'industrie disent qu'il est préférable de conserver tous vos styles CSS que vous allez
implémenter dans un seul fichier SCSS.

Certaines bibliothèques comme EmotionsJS & Glamorous vous permettront d'écrire du CSS en JavaScript.

Si vous pensez que certains noms CSS pourraient entrer en collision, l'utilisation de préfixes globaux peut aider à éviter cela. Mais si la taille de
votre projet augmente, cela pourrait ne pas être une approche pratique.

### Ne commentez que si nécessaire

Lorsque cela est nécessaire, un développeur doit ajouter des commentaires aux lignes de code. Les commentaires sont l'une des pratiques les plus
efficaces lors de l'utilisation de React Architecture.

Ça aidera :
- Gardez les codes plus faciles à lire.
- Si vous avez l'intention de modifier le code ultérieurement, cela peut aider à éviter un conflit entre le commentaire et le code.

### Utilisez des majuscules pour les noms de composants

La majorité des développeurs React utilisent JSX (une extension JavaScript). Si vous envisagez également de l'utiliser, vous devez vous assurer que les
noms des composants que vous créez doivent commencer par des lettres majuscules.

Par exemple, vous nommerez les composants comme « SelectButton » et non « selectbutton ». En procédant ainsi, le JSX les identifiera automatiquement de
manière unique à partir des balises HTML par défaut.

Les premières versions de React avaient une liste de tous les noms intégrés pour les différencier des noms personnalisés. Comme la liste devait être
mise à jour régulièrement, les majuscules sont devenues la nouvelle norme.

Cependant, c'est spécifiquement si vous optez pour JSX. Si vous utilisez une autre langue, vous pouvez utiliser des lettres minuscules. Mais gardez à
l'esprit que cela peut aider à réduire la réutilisabilité des composants au-delà de la portée de votre projet.

### Utilisez l'outil Bit de React

![Use React’s Bit tool](https://www.communicationcrafts.com/wp-content/uploads/2021/06/CC-Blog_React-Architecture-Best-Practices-and-Tips-from-Experts_inner-banner_3.jpg)

L'outil Bit est l'un des meilleurs outils React qui vous aidera à mieux organiser facilement les composants React dans votre projet.

Cet outil contribue également à augmenter la réutilisabilité du code, tout en encourageant les équipes à collaborer pour créer des composants.

### Utilisez des bibliothèques d'extraits (snippet)

Les meilleures pratiques d'architecture React impliquent également l'utilisation d'extraits de code.

Lorsque vous utilisez des extraits de code, il vous sera plus facile, en tant que développeur, de suivre la dernière syntaxe. Vous pourrez également
garder le code relativement exempt de bogues, c'est pourquoi en tant que développeur, c'est l'une des pratiques de pari React à suivre.

Certaines bibliothèques d'extraits React que vous pouvez utiliser sont :
- ES7 React
- Redux
- JS Snippets
